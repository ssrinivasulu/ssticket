Filtering data using Outline View

Learn how to filter data and view the most relevant records.

Besides displaying Document structure, Outline View is also used to quickly find and filter specific Documents.

The most basic filtering can be done using the Exists field. Checking it for a specific Field ensures that only Documents which have the Field defined are displayed in Object Detail.

Note: Hover over editable Filter field to see possible filtering options.

The following data type Fields can be filtered further:

Data type Filter examples
Date
  • 12/31/1999 - all with value 12/31/1999
  • !12/31/1999 - all except for value 12/31/1999
  • 2000, 2005, 2007 - all with year equal to 2000, 2005, 2007
  • > 12/31/1999 - more recent than 12/31/1999
  • >= 12/31/1999 - equal, or more recent than 12/31/1999
  • 12/31/1999-11/30/2014 - all from 12/31/1999 to 11/30/2014
  • !12/31/1999-11/30/2014 - all except from 12/31/1999 to 11/30/2014
  • 2001-2005, 2008-2010 - all from 2000 to 2005 and from 2008 to 2010
  • 2000-2010, !2006-2008 - all from 2000 to 2010 except for value from 2006 to 2008
  • 2000-2010, !2006, !2007 - values from 2000 to 2010 except for 2006 and 2007
Number
  • >0 - positive values
  • >=0 - positive values and 0
  • 0-5 - values from 0 to 5
  • !6-8 - all values except for 6 to 8
  • 0-5, 8-10 - values from 0 to 5 and from 8 to 10
  • 0-5, !1-3 - values 0, 4, 5
  • 0-10, !6, !7 - values 0, 1, 2, 3, 4, 5, 8, 9, 10
String

Can be filtered using regular expressions the same way as in MongoDB shell.
