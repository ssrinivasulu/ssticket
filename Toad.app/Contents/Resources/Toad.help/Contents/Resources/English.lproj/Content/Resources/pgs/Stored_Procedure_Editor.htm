Stored Procedure Editor

Take advantage of Toad Extension PL/SQL support.

Note: Editor is available only for Oracle databases.

The Editor serves primarily for developing and storing PL/SQL scripts.

A new Editor can be opened via View Menu | Editors | Show Stored Procedure Editor.

Editor options

The options are located on Editor Toolbar.

Button Option Description
Store in Database and Compile Stores the script in database as a compilable script, which can then be Run.
Store in Database and Compile with Debug Stores the script in database as a compilable script, which can then be Run. Also allows debugging of the script.
Run Runs the stored script.
Run Again Runs the stored script. Uses the same parameters as the last Run.
Select Connection Determines the connection used for storing/compiling/running scripts.

Additional Editor right-click options

Option Description
Make Code Statement Selecting an PL/SQL statement and using this option will create a Java code that creates a new query and appends the selected statement to it. The code is stored in clipboard.
Strip Code Statement Selecting a Java code which contains PL/SQL statement and using this option deletes the Java code and keeps the statement. The statement is stored in clipboard.

Hint Window

When you hover over a database object name, Hint Window displays and provides you with information about the object. Hover over the bottom bar of Hint Window to display up to three additional options.

Button Option Description
Open Declaration The object is opened in Object Detail.
Show Script The object SQL code is shown in a new worksheet/editor.
Dependencies The object dependencies are shown in Dependencies View.
